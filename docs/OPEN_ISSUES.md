# 项目遗留问题与后续优化清单

更新时间：2026-03-08

## 1. 文档定位

- 本文档用于集中记录当前项目仍未闭环的问题、文档缺口、验证缺口与后续优化建议。
- 它不替代 `docs/STATUS.md` 的状态总览，而是补充“下一步应该做什么、为什么做、优先级如何排”。
- 目标是让后续开发不必从 README、状态文档和历史对话中重新拼接上下文。

## 2. 当前判断

- 项目主线功能已经可用，核心 MVP 不处于“不可交付”状态。
- 高级同步策略已经完成 V1 闭环：后端 API、输出聚合接入、管理后台配置区、preview 展示、策略指标、后端接口级测试与本地回归基线均已落地。
- 当前主要问题已经从“缺功能”转为“缺进一步产品化能力与前端可维护性优化”。
- 因此，后续工作应优先围绕“验证闭环”和“可维护性”推进，而不是继续无边界扩展新功能。

## 4. 仍然存在的问题

### P1：高价值优化

#### 4.1 前端主文件过大，维护成本升高

问题：

- `web/src/App.jsx` 已承担登录、上游、节点、设置、策略、备份、日志等全部视图逻辑。

影响：

- 后续再扩展策略功能、日志功能或备份功能时，改动风险和阅读成本都会变高。

建议动作：

- 按功能拆分为最少 3~5 个组件，例如：
  - `SettingsPanel`
  - `StrategyPanel`
  - `BackupPanel`
  - `LogsPanel`

#### 4.2 策略 UX 仍可继续打磨

问题：

- 当前可用，但缺少“保存成功提示 / 预览已过期提示 / 未保存变更提醒 / 风险提示”等细节。

影响：

- 管理员可以完成配置，但在反复调参时容易缺乏操作反馈。

建议动作：

- 增加轻量成功提示
- 标记 preview 是否基于当前未保存配置
- 在 `priority_override` 模式下增加“会丢弃冲突节点”的风险说明

### P2：产品化增强项

#### 4.3 更细粒度策略能力仍未实现

当前未做：

- 按输出目标分别配置策略
- 单上游独立规则
- 单节点例外规则
- 更灵活的去重维度

说明：

- 这些都属于产品化增强，不建议在当前阶段优先于验证闭环和可维护性。
