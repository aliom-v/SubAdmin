# 项目遗留问题与后续优化清单

更新时间：2026-03-08

## 1. 文档定位

- 本文档用于集中记录当前项目仍未闭环的问题、文档缺口、验证缺口与后续优化建议。
- 它不替代 `docs/STATUS.md` 的状态总览，而是补充“下一步应该做什么、为什么做、优先级如何排”。
- 目标是让后续开发不必从 README、状态文档和历史对话中重新拼接上下文。

## 2. 当前判断

- 项目主线功能已经可用，核心 MVP 不处于“不可交付”状态。
- 高级同步策略已经完成 V1 闭环：后端 API、输出聚合接入、管理后台配置区、preview 展示、策略指标、后端接口级测试与本地回归基线均已落地。
- 当前主要问题已经从“缺功能”转为“缺远端验证证据补齐与进一步产品化增强”。
- 因此，后续工作应优先围绕“验证闭环”和“可维护性”推进，而不是继续无边界扩展新功能。

## 4. 仍然存在的问题

### P1：高价值优化

#### 4.1 远端 / CI 证据仍待补齐

问题：

- 当前本地回归与构建基线已经形成，但还缺少首次远端 / CI 全流程执行后的证据回填。

影响：

- 本地可验证并不等于远端留痕完整，后续交接、审计或回顾时仍需要额外手工补证据。

建议动作：

- 触发一次远端 / CI 全流程
- 留存 acceptance / smoke / build 相关 artifact
- 将执行时间和证据位置回填到 `docs/PHASE4_REGRESSION.md` 与 `docs/STATUS.md`

### P2：产品化增强项

#### 4.2 更细粒度策略能力仍未实现

当前未做：

- 按输出目标分别配置策略
- 单上游独立规则
- 单节点例外规则
- 更灵活的去重维度

说明：

- 这些都属于产品化增强，不建议在当前阶段优先于验证闭环和可维护性。

## 5. 文档层面的剩余缺口

当前文档主干已经基本补齐。

仍需补的是“随实现变化持续更新内容”，而不是新增新的说明文档：

1. 后续新增更细粒度策略能力后的文档更新
2. 远端 / CI 首次跑通后的证据与时间回填

当前不建议继续增加的有：

- 不建议再写新的“大而全规划文档”
- 不建议继续新增仅用于堆叠历史背景的归档类文档
- 不建议把实现细节过度堆到 `README.md`

1. 补齐远端 / CI 证据
2. 继续做更细粒度策略增强
3. 若前端继续增长，再按需抽离 hooks / shared utils
